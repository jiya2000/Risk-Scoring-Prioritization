\section{4. Summary and Background of the Invention}

\subsection{4.1 Background}

Anti-Money Laundering (AML) compliance teams at financial institutions receive between 50,000 and 200,000 transaction alerts per week. Legacy rule-based alert systems produce 95\%+ false positive rates, resulting in ``alert fatigue.''

Existing ML-based AML systems suffer from three fundamental architectural gaps:

\textbf{Gap 1 --- Static ensemble weights:} All prior ensemble methods (including US20220405860) use globally learned or fixed weights to combine heterogeneous signals. A smurfing-ring account and an isolated account receive identical weights.

\textbf{Gap 2 --- Temporally unaware graph centrality:} Prior graph-based AML methods (including US11640609B1) compute centrality features on discretely time-binned transaction graphs using fixed-step matrix propagation of risk vectors ($x = A^p z$). Edge weights may incorporate recency as a static binning factor, but temporal decay is not embedded continuously in the centrality iteration itself. Burst-pattern transactions receive the same weight as ordinary patterns. Cyclic fund transfers are penalized uniformly.

\textbf{Gap 3 --- No precision guarantee during component failures:} Production AML systems have no mechanism to guarantee that the fraud prioritization queue maintains a minimum quality level when individual components fail. Prior art (US20210174258A1) monitors component health and emits alerts but does not route computation through alternative execution paths with pre-validated precision scores.

\subsection{4.2 Summary of the Invention}

This invention presents three distinct novel technical contributions, each independently patentable:

\textbf{Contribution 1 --- Temporal-Decay PageRank (TD-PageRank):}
A modified PageRank algorithm where the exponential temporal decay factor $\lambda = \ln(2)/\text{half\_life\_days}$ is embedded directly inside the power iteration formula. Additionally: (a) burst-velocity amplification; (b) log-amount normalization; (c) directional SCC penalty (collector 0.25$\times$, distributor 0.5$\times$, balanced 0.375$\times$); (d) dormant node suppression at 0.1$\times$ max score.

\textbf{Contribution 2 --- Topology-Adaptive Fusion with TopologyAttentionGate:}
A per-account dynamic weight computation system where a 6-dimensional topology vector is extracted from each account's 2-hop ego-network and passed through a TopologyAttentionGate --- a multiplicative gating neural network that amplifies the graph-signal weight for topologically active accounts.

\textbf{Contribution 3 --- Adaptive Degradation Controller with Precision Budget:}
A system that monitors 5 pipeline components and routes scoring computation through one of 8 pre-evaluated execution paths, each with empirically measured Precision@50. Sub-mechanisms: (a) PrecisionDriftDetector (EMA-smoothed KL-divergence); (b) AdaptivePrecisionBudget (self-calibrating threshold).

\section{6. Working Principle of the Invention (In Brief)}

\subsection{Innovation 1: Temporal-Decay PageRank (TD-PageRank)}

The standard PageRank power iteration computes $r[v] = (1-d)/N + d \times \sum (r[u] \times w(u,v) / \sum w(u,k))$. In our invention, edge weight $w(u,v)$ is replaced by the temporally-decayed weight:

\begin{lstlisting}
w_temporal(e) = w_original(e) * exp(-lambda * Edge_Age(e)) * burst_multiplier(sender)
\end{lstlisting}

where:
\begin{itemize}
  \item $w_{\text{original}}(e)$ = transaction amount (log1p scaled when enabled)
  \item $\lambda = 0.693 / \text{half\_life\_days}$ (default half\_life = 7 days)
  \item $\text{Edge\_Age}(e) = (\text{reference\_date} - \text{edge.Date}).\text{days}$
  \item $\text{burst\_multiplier}(\text{sender}) = 1 + (\text{window\_count} / \text{total\_count})$ if sender sent $>$5 transactions in last 3 days, else 1.0
\end{itemize}

\subsection{Innovation 2: Topology-Adaptive Fusion with TopologyAttentionGate}

For each transaction, the sender account's 2-hop ego-network is extracted and a 6-dimensional Topology Vector is computed:

\begin{lstlisting}
gate       = sigmoid(W_gate * topology_vector)
gate_score = mean(gate)
gate_amp   = 1.0 + 2.0 * gate_score       # amplifier in [1.0, 3.0]
base_w     = softmax(W_base * topology_vector)
w_graph    = base_w[1] * gate_amp
w_final    = clamp(w/sum(w), 0.05, 0.90)
\end{lstlisting}

The fused score: $\text{score} = w_{\text{ml}} \times S_{\text{ml}} + w_{\text{graph}} \times S_{\text{graph}} + w_{\text{rules}} \times S_{\text{rules}}$

\subsection{Innovation 3: Adaptive Degradation Controller}

Every 10 seconds, 3-metric HealthVectors are computed for each component. State transitions: HEALTHY $\to$ DEGRADED after 2 consecutive unhealthy cycles; DEGRADED $\to$ HEALTHY after 3 consecutive healthy cycles; flap protection locks COOLDOWN for 5 minutes.

Path selection: $\text{select\_path}(\text{healthy\_set}) = \arg\max P@50$ such that $\text{required\_components} \subseteq \text{healthy\_set}$ AND $P@50 \geq \text{AdaptiveBudget.current}()$.

\subsection{7.2 Innovation 1: TD-PageRank Engine}

\textbf{Class:} \texttt{TDPageRankEngine}\\
\textbf{Parameters:} \texttt{half\_life\_days=7.0}, \texttt{damping=0.85}, \texttt{cycle\_penalty=0.5}, \texttt{max\_iter=100}, \texttt{tol=1e-6}, \texttt{burst\_window\_days=3}

\textbf{Algorithm --- Full Power Iteration:}

\begin{lstlisting}
Initialize: r[v] = 1/N for all nodes v

For t = 1 to max_iter:
  For each node v:
    dangling_contribution = damping * (sum r[u] for all dangling u) / N
    r_new[v] = (1-damping)/N + dangling_contribution
              + damping * sum_{u->v} (r[u] * w_temporal(u,v) / sum_{u->k} w_temporal(u,k))
  if L1(r_new - r) < tol: converged = True; STOP
  r = r_new

Post-processing:
  For each node v in cycle_nodes (SCC size > 2):
    intra_ratio = sum intra-SCC weight / sum total weight
    if intra_ratio > 0.80:
      if asymmetric_scc_penalty:
        if in_scc_weight > out_scc_weight:
          r[v] *= cycle_penalty * 0.5    # collector: 0.5 * 0.5 = 0.25x
        elif out_scc_weight > in_scc_weight:
          r[v] *= cycle_penalty * 1.0    # distributor: 0.5 * 1.0 = 0.50x
        else:
          r[v] *= cycle_penalty * 0.75   # balanced: 0.5 * 0.75 = 0.375x
      else:
        r[v] *= cycle_penalty            # symmetric (default)

  For dormant nodes (all incident edges > 30 days):
    r[v] = min(r[v], 0.1 * max(r))

  Normalize: r_normalized[v] = (r[v] - min(r)) / (max(r) - min(r))
\end{lstlisting}

\textbf{Output:} \texttt{TDPageRankResult} containing: \texttt{scores}, \texttt{normalized\_scores}, \texttt{cycle\_member}, \texttt{decay\_impact}, \texttt{converged}, \texttt{iterations}, \texttt{reference\_date}


\subsection{7.3 Innovation 2: Topology-Adaptive Fusion}

\textbf{EgoNetworkExtractor} computes 6-dimensional \texttt{TopologyVector}:
\begin{itemize}
  \item $\text{edge\_density} = |E| / (|V| \times (|V|-1))$
  \item diameter = longest shortest path in undirected ego-network
  \item $\text{avg\_clustering}$ = mean clustering coefficient
  \item $\text{degree\_asymmetry} = \text{Var}(\text{in\_degree}) / \max(\text{Var}(\text{out\_degree}), 10^{-10})$
  \item $\text{component\_ratio} = |\text{largest WCC}| / |V|$
  \item $\text{tx\_velocity\_ratio} = \text{recent\_edges}(\leq 7\text{ days}) / \text{total\_edges}$
\end{itemize}

\textbf{TopologyAttentionGate Architecture:}
\begin{lstlisting}
Layer 1: Linear(5, 5) -> Sigmoid        [gate layer]
Layer 2: Linear(5, 32) -> ReLU -> Linear(32, 3)   [base weight allocator]
Gate computation:
  gate_activity = mean(sigmoid_output)
  gate_amplifier = 1.0 + 2.0 * gate_activity   [in (1.0, 3.0)]
  base_weights = softmax(base_allocator output)
  w_graph *= gate_amplifier
  weights = clamp(w/sum(w), 0.05, 0.90)
\end{lstlisting}

\textbf{Fallback Logic:} Static weights (0.70, 0.15, 0.15) applied when: graph unavailable, topology extraction $>$200ms, network inference $>$50ms, NaN/Inf in outputs, or any weight outside [0.05, 0.90].

\textbf{Isolation Rule:} When ego-network has $<$3 nodes: $w_{\text{ml}}$ forced $\geq$ 0.70.

\section{9. What Aspect(s) of the Invention Need(s) Protection?}

The following aspects constitute novel, non-obvious technical contributions that satisfy \S~101 patent eligibility:

\subsection{Claim 1 --- Temporal-Decay PageRank Algorithm with Burst-Velocity Amplification and Directional SCC Penalty}

\textbf{What must be protected:}

\begin{enumerate}[label=1.\arabic*]
  \item The method of embedding exponential temporal decay $\exp(-\lambda \times \text{Edge\_Age})$ directly within the PageRank power iteration transition matrix, as opposed to computing centrality via fixed-step risk-vector propagation on discretely time-binned adjacency matrices (the approach of US11640609B1). This is the core algorithmic novelty.
  \item The burst-velocity amplification mechanism: a per-edge weight multiplier $(1 + \text{window\_count}/\text{total\_count}) \geq 1.0$ applied to sender nodes exhibiting transaction surges within a configurable time window.
  \item The log-amount normalization: replacing raw transaction amounts with $\log(1 + \text{amount})$ before temporal weighting.
  \item The directional SCC penalty: asymmetric cycle dampening --- collectors (0.25$\times$), distributors (0.5$\times$), balanced (0.375$\times$). Distinct from all prior art which applies uniform cycle penalties.
  \item The dormant node suppression: capping scores of nodes with all edges exceeding a configurable age threshold (default 30 days) at 0.1$\times$ the maximum score.
\end{enumerate}

\textbf{Scope of protection:} Method claims covering the combination of (1.1) through (1.5) as applied to directed financial transaction graphs with timestamped edges.

\subsection{Claim 2 --- Topology-Conditioned Multiplicative Attention Gate for Heterogeneous Signal Fusion}

\textbf{What must be protected:}

\begin{enumerate}[label=2.\arabic*]
  \item Computing a per-account 5- or 6-dimensional Topology Vector from the account's 2-hop ego-network.
  \item The TopologyAttentionGate architecture: multiplicative gating on the ensemble weight dimension (not additive attention on feature space). Gate amplifier range [1.0, 3.0].
  \item The complete per-account weight computation pipeline: Topology Vector $\to$ TopologyAttentionGate $\to$ clamped weights [0.05, 0.90] $\to$ fused score.
  \item The isolation safety floor: $w_{\text{ml}} \geq 0.70$ when ego-network $<$ 3 nodes.
  \item The complete fallback chain: 6 distinct fallback conditions triggering static weights (0.70, 0.15, 0.15).
\end{enumerate}

\textbf{Scope of protection:} System and method claims covering (2.1)--(2.5) as a per-account dynamic ensemble weighting mechanism.

\subsection{Claim 3 --- Precision-Budget-Constrained Adaptive Degradation Controller}

\textbf{What must be protected:}

\begin{enumerate}[label=3.\arabic*]
  \item Routing ML scoring computation through a pre-computed execution path table with empirically measured Precision@K values, constrained to paths meeting a configurable precision budget.
  \item The PrecisionDriftDetector: EMA-smoothed reference distribution, KL-divergence drift detection, configurable threshold alerts.

\subsection{Independent Claim 3 --- Adaptive Degradation Controller with Precision Budget}

A system for precision-budget-constrained adaptive degradation control in a multi-component ML scoring pipeline, comprising:

\begin{enumerate}[label=(\alph*)]
  \item a precision-budget routing table comprising a plurality of pre-evaluated execution paths, each path annotated with an empirically measured Precision@K value obtained from offline evaluation, wherein path selection is constrained to paths meeting a configurable precision budget rather than routing based on latency or availability metrics;
  \item a PrecisionDriftDetector that maintains an EMA-smoothed reference distribution of scoring pipeline outputs, computes KL-divergence between a live output distribution and said reference distribution, and raises a drift alert when said KL-divergence exceeds a configurable threshold, thereby enabling proactive detection of precision degradation before it manifests in measured Precision@K values;
  \item an AdaptivePrecisionBudget that implements a self-calibrating minimum precision threshold, said threshold tightening by a first increment when rolling shadow evaluation history shows performance consistently exceeding a baseline by a first margin, relaxing by a second decrement when performance is borderline, and tightening by a third increment when drift KL-divergence exceeds a drift threshold, with said threshold bounded by a configurable floor and ceiling;
  \item a health-monitoring state machine providing: detection of DEGRADED state after a first number of consecutive unhealthy monitoring cycles, restoration of HEALTHY state after a second number of consecutive clean cycles, flap protection entering COOLDOWN state after more than a third number of state transitions within a configurable time window, shadow evaluation with escalation when measured precision falls below budget minus a margin, and a stale queue policy with configurable maximum age; and
  \item integration of said PrecisionDriftDetector with said AdaptivePrecisionBudget, wherein drift KL-divergence is passed as an additional input to the budget computation, tightening the effective budget when drift exceeds a drift integration threshold, thereby creating a two-signal early-warning system that pre-tightens quality requirements before precision actually degrades.
\end{enumerate}

\subsection{Independent Claim 1 --- Topology-Adaptive Fusion}

A computer-implemented method for adaptively fusing heterogeneous risk signals for anti-money laundering detection, comprising:

\begin{enumerate}[label=(\alph*)]
  \item extracting, for each account in a transaction graph, an N-dimensional topology vector from a K-hop ego-network surrounding said account, wherein said topology vector encodes structural properties including degree centrality, clustering coefficient, edge density, degree asymmetry, connected component ratio, and local connectivity patterns;
  \item passing said topology vector through a multiplicative attention gate comprising a learned sigmoid gating layer that produces a per-dimension activity vector and a scale factor, wherein said gate computes a gate amplifier value as a function of mean gating activity, said gate amplifier modulating a graph-signal weight independently of feature-space attention;
  \item computing per-account fusion weights by multiplying a base weight allocation, produced by a softmax weight allocator conditioned on said topology vector, by said gate amplifier, thereby producing topology-conditioned weights that vary as a function of local graph structure and that are computed independently for each account rather than learned globally at training time; and
  \item fusing a machine learning score, a graph intelligence score, and a symbolic rule score using said per-account fusion weights to produce a final risk score for each account, wherein said fusion weights are clamped to a minimum of 0.05 and a maximum of 0.90 per signal and normalized to sum to unity.
\end{enumerate}

\subsection{Independent Claim 2 --- Temporal-Decay PageRank (TD-PageRank)}

A computer-implemented method for scoring nodes in a financial transaction graph, comprising:

\begin{enumerate}[label=(\alph*)]
  \item computing, for each directed edge in said transaction graph, a temporal weight $w = \text{amount} \times \exp(-\lambda \times \text{age})$, wherein $\lambda$ is a decay constant derived from a configurable half-life parameter and age is measured in days from a reference date;
  \item constructing a weighted directed graph using said temporal weights as edge weights;
  \item detecting burst-velocity nodes by identifying accounts whose transaction count within a sliding window exceeds a burst threshold, and amplifying scores for said nodes by a burst multiplier proportional to the ratio of window transactions to total transactions;
  \item identifying strongly connected components (SCCs) of size greater than 2, and applying a directional SCC penalty comprising asymmetric multipliers based on flow direction: collector nodes receiving 0.25$\times$ base penalty, distributor nodes receiving 0.50$\times$ base penalty, and balanced nodes receiving 0.375$\times$ base penalty; and
  \item computing a modified PageRank score for each node using said temporal weights, burst amplification, and directional penalties, wherein transactions with greater age contribute exponentially less influence to node scores.
\end{enumerate}
