\section{4.\ Summary and Background of the Invention}
% ══════════════════════════════════════════════════════════════════════

\subsection{4.1 Background}

AML compliance teams receive 50,000--200,000 transaction alerts per week. Legacy rule-based
systems produce \(>\)95\% false-positive rates. Existing ML-based systems suffer three
fundamental architectural gaps:

\textbf{Gap 1 --- Static ensemble weights:} All prior ensemble methods (including
US20220405860) use globally-learned weights applied uniformly. A smurfing-ring account
and an isolated account both receive the same fixed weights, which is structurally incorrect.

\textbf{Gap 2 --- Temporally unaware graph centrality:} Prior graph-based methods compute
centrality on discretely time-binned snapshots. Temporal decay is not embedded continuously
in the iteration. Burst patterns (smurfing hallmark) receive equal weight to ordinary patterns.

\textbf{Gap 3 --- No precision guarantee during failures:} Production systems have no mechanism
to guarantee queue quality when components fail. Prior art (US20210174258A1) monitors health
but does not route through precision-validated alternative paths.

\textbf{Gap 4 (New) --- Static heuristics and manual features:} Existing SCC penalties rely on
three hard-coded archetypes. Topology representation is manually engineered. Precision monitoring
relies on offline-computed P@50 assumptions that become stale under concept drift.

\subsection{4.2 Summary of Original Three Innovations}

\textbf{Innovation 1 --- Temporal-Decay PageRank (TD-PageRank):} Exponential decay
$w_\text{temporal}=\text{amount}\times e^{-\lambda\times\text{EdgeAge}}$ embedded
\emph{inside} the power iteration, plus burst-velocity amplification, directional SCC penalty,
log-amount normalisation, and dormant-node suppression.

\textbf{Innovation 2 --- Topology-Adaptive Fusion with TopologyAttentionGate:}
Per-account 6-metric topology vector extracted from 2-hop ego-network, passed through a
\emph{multiplicative} attention gate (distinct from additive attention) that amplifies
the graph-signal weight proportionally to network density.

\textbf{Innovation 3 --- Adaptive Degradation Controller:} Routes scoring through one of 8
pre-evaluated execution paths (each with empirically measured P@50), with
\texttt{PrecisionDriftDetector} (EMA-KL divergence) and self-calibrating
\texttt{AdaptivePrecisionBudget} guaranteeing P@50\,$\geq$\,0.60 across all operational modes.

\subsection{4.3 Architecture Hardening Innovations (New)}

The architecture-hardening layer adds four additional surfaces, all guarded by
\texttt{ArchitectureHardeningConfig} flags defaulting to \texttt{False} (fully backward-compatible):

\textbf{Hardening Surface 1 --- Learnable SCC Penalty:} Replaces the 3-archetype static
heuristic with a trainable 2-layer MLP (\texttt{LearnableSCCPenalty}) that maps 5 intra-SCC
flow features to penalty multipliers in [0.1,\,1.0], retrained without code changes via
\texttt{train\_penalty()} and updated atomically via \texttt{ResourceManager.atomic\_model\_swap()}.

\textbf{Hardening Surface 2 --- GNN Topology Embedding:} Replaces the manual 5-dimensional
\texttt{TopologyVector} with \texttt{TopologyEmbeddingNetwork} (2 GCNConv layers, 16-dimensional
embedding) plus \texttt{IsolationDetector} (learned continuous isolation score replacing the
hard-coded w\_ml\,$\geq$\,0.70 threshold) and \texttt{DeepTopologyAttentionGate}
(2+ hidden layers, dropout, multiplicative gating preserved).

\textbf{Hardening Surface 3 --- Online Precision Recalibration:} Replaces static offline
P@50 assumptions with \texttt{OnlinePrecisionMonitor} (rolling 500-sample window, P@50
recomputed live), \texttt{EnhancedConceptDriftDetector} (KL-divergence + precision jointly
classify: \textit{no\_drift} / \textit{benign\_shift} / \textit{precision\_degraded}), and
\texttt{EnhancedAdaptivePrecisionBudget} (weighted blending: 0.7$\times$online + 0.3$\times$shadow).

\textbf{Hardening Surface 4 --- \S101 Resource Management Anchoring:}
\texttt{ResourceManager} records every execution-path switch with concrete resource metrics:
memory allocation delta, computation-time delta, and active processing-unit count ---
anchoring all patent claims to system-level technological improvements that survive
\textit{Alice Corp.\ v.\ CLS Bank} abstract-idea challenges.

\section{6.\ Working Principle of the Invention (In Brief)}
% ══════════════════════════════════════════════════════════════════════

\subsection{Innovation 1: Temporal-Decay PageRank (TD-PageRank)}

Edge weight $w_\text{temporal}(e)$ replaces $w_\text{original}$ in the power iteration:
\[
  w_\text{temporal}(e) = w_\text{original}(e)\times e^{-\lambda\times\text{EdgeAge}(e)}
                         \times \text{burst\_mult}(\text{sender})
\]
where $\lambda = \ln 2 / \text{half\_life\_days}$ (default 7 days). Post-convergence:
asymmetric SCC penalties (collector 0.25$\times$, distributor 0.50$\times$, balanced 0.375$\times$)
and dormant-node capping (score\,$\leq$\,0.1$\times$max) are applied.

\subsection{Innovation 2: Topology-Adaptive Fusion with TopologyAttentionGate}

\begin{lstlisting}
gate        = sigmoid(W_gate x topology_vector)   # per-dim activity in (0,1)
gate_score  = mean(gate)                          # scalar topology activity
gate_amp    = 1.0 + 2.0 * gate_score             # amplifier in [1.0, 3.0]
base_w      = softmax(W_base x topology_vector)   # base allocation
w_graph    *= gate_amp                            # amplify graph weight only
w_final     = clamp(w / sum(w), 0.05, 0.90)      # clamp-normalise x3
fused_score = w_ml*S_ml + w_graph*S_graph + w_rules*S_rules
\end{lstlisting}

\subsection{Innovation 3: Adaptive Degradation Controller}

Every 10 s: 3-metric HealthVector per component $\to$ state machine (HEALTHY/DEGRADED/COOLDOWN)
$\to$ \texttt{select\_path}(healthy\_set) $= \arg\max P@50$ s.t.\ required\_components $\subseteq$
healthy\_set AND $P@50 \geq \texttt{AdaptiveBudget.current()}$.
\texttt{PrecisionDriftDetector} (EMA-smoothed KL-divergence) provides proactive early warning.

\subsection{Innovation 4: Architecture Hardening Layer}

\texttt{ArchitectureHardeningConfig} controls all flags (all default = \texttt{False}).
When enabled: \texttt{LearnableSCCPenalty} MLP replaces static 3-archetype heuristic via
\texttt{train\_penalty()} + \texttt{ResourceManager.atomic\_model\_swap()};
\texttt{TopologyEmbeddingNetwork} (2$\times$GCNConv, 16-d embedding) + \texttt{IsolationDetector}
replace manual \texttt{TopologyVector} with 200\,ms timeout/fallback;
\texttt{DeepTopologyAttentionGate} (2+ hidden layers, $\geq$64 hidden dim, dropout) replaces single-layer gate;
\texttt{OnlinePrecisionMonitor} (rolling 500-sample window) + \texttt{EnhancedConceptDriftDetector}
+ \texttt{EnhancedAdaptivePrecisionBudget} (0.7$\times$online + 0.3$\times$shadow blending) replace
static offline assumptions; \texttt{ResourceManager} logs every path switch with memory delta,
computation time, and active-unit count.

\subsection{7.2 Innovation 1: TD-PageRank Engine (\texttt{models/td\_pagerank.py})}

\begin{figure}[H]
\centering
\includegraphics[width=0.72\textwidth]{artifacts/fig02_tdpagerank_flowchart.png}
\caption{TD-PageRank algorithm flowchart: temporal weighting, burst amplification,
power iteration, directional SCC penalty, and dormant-node suppression.}
\label{fig:tdpr}
\end{figure}

\textbf{Class:} \texttt{TDPageRankEngine}. Key parameters: \texttt{half\_life\_days=7.0},
\texttt{damping=0.85}, \texttt{cycle\_penalty=0.5}, \texttt{max\_iter=100}, \texttt{tol=1e-6},
\texttt{asymmetric\_scc\_penalty=False}, \texttt{use\_learnable\_penalty=False}.

\textbf{Power Iteration (full algorithm):}
\begin{lstlisting}
r[v] = 1/N  for all v
for t in range(max_iter):
    dangling = damping * sum(r[u] for u in dangling_nodes) / N
    r_new[v] = (1-damping)/N + dangling
               + damping * SUM_{u->v} r[u]*w_temporal(u,v) / SUM_{u->k} w_temporal(u,k)
    if L1(r_new - r) < 1e-6: CONVERGED; break
    r = r_new

# Post-processing:
for node in cycle_nodes (SCC size > 2, intra_ratio > 0.80):
    if asymmetric_scc_penalty:
        if collector: r[node] *= 0.25  # inflow-dominant
        elif distrib: r[node] *= 0.50  # outflow-dominant
        else:         r[node] *= 0.375 # balanced
for dormant_node (all edges > 30 days):
    r[node] = min(r[node], 0.1 * max(r))
r_normalized = (r - min(r)) / (max(r) - min(r))
\end{lstlisting}

\subsection{7.3 Innovation 2: Topology-Adaptive Fusion (\texttt{models/adaptive\_fusion.py})}

\begin{figure}[H]
\centering
\includegraphics[width=0.92\textwidth]{artifacts/fig03_topology_attention_gate.png}
\caption{TopologyAttentionGate neural architecture: multiplicative gating on the weight
dimension (not feature-space attention).}
\label{fig:gate}
\end{figure}

\textbf{EgoNetworkExtractor} computes a 6-dimensional \texttt{TopologyVector}:
edge density $|E|/(|V|(|V|-1))$; graph diameter; average clustering coefficient;
degree asymmetry $\text{Var}(\text{in\_deg})/\max(\text{Var}(\text{out\_deg}),\epsilon)$;
component ratio; transaction velocity ratio ($\leq$7-day edges / total edges).

\textbf{TopologyAttentionGate:} Layer 1: Linear(5$\to$5) + Sigmoid (gate);
Layer 2: Linear(5$\to$32) + ReLU + Linear(32$\to$3) (base allocator).
Gate amplifier $= 1.0 + 2.0\times\overline{\text{gate}}\in[1.0,3.0]$.
$w_\text{graph}$ only is multiplied by amplifier; all weights then clamped-normalised
($\times$3 iterations) to $[0.05,\,0.90]$.

\textbf{Fallback:} Static weights $(0.70, 0.15, 0.15)$ on 6 conditions:
graph unavailability, topology timeout $>$200\,ms, inference timeout $>$50\,ms,
NaN/Inf output, weight range violation, ego-network isolation ($<$3 nodes $\Rightarrow$
$w_\text{ml} \geq 0.70$).

\subsection{7.5 Architecture Hardening: Learnable SCC Penalty
          (\texttt{models/td\_pagerank.py --- LearnableSCCPenalty})}

A 2-layer MLP (input\_dim=5, hidden\_dim=32) maps 5 intra-SCC flow features
$[\text{intra\_inflow},\,\text{intra\_outflow},\,\text{weight\_ratio},\,\text{scc\_size},\,
\text{node\_degree}]$ to penalty multipliers via sigmoid scaling:
$\text{penalty} = 0.1 + 0.9\times\sigma(\text{MLP}(x))$, guaranteeing output $\in[0.1,\,1.0]$
for any input. Training uses mandatory temporal splits (\texttt{train\_penalty()} enforces
chronological split at \texttt{temporal\_split\_date}).

\subsection{7.6 Architecture Hardening: GNN Topology Embedding
          (\texttt{models/adaptive\_fusion.py --- TopologyEmbeddingNetwork})}

Two GCNConv message-passing layers (hidden\_dim=32) followed by global mean pooling
and a linear projection to 16-dimensional embedding. Node features: 4-dimensional
$[\text{in\_degree},\,\text{out\_degree},\,\text{amount\_sum},\,\text{edge\_count}]$.
Timeout enforced at 200\,ms; fallback to manual \texttt{TopologyVector} on timeout or PyG
unavailability. \texttt{IsolationDetector}: 3-layer MLP $\to$ sigmoid $\in[0,1]$;
\texttt{compute\_fusion\_weights()} applies:
$w_\text{ml}^\text{new} = w_\text{ml} + \text{isolation}\times(0.90 - w_\text{ml})$
(monotonically non-decreasing, capped at 0.90).

\subsection{7.7 Architecture Hardening: DeepTopologyAttentionGate
          (\texttt{models/adaptive\_fusion.py --- DeepTopologyAttentionGate})}

MLP with $\geq$2 hidden layers (hidden\_dim$\geq$64) + Dropout + final linear(3) + softmax.
Topology activity $= \sigma(\|e\|_2)$; graph-signal weight amplified by
$1 + 2.0\times\text{activity}$ before iterative clamp-normalise.
Inference budget: $<$50\,ms. Raises \texttt{ValueError} if \texttt{num\_hidden\_layers}$<$2
or \texttt{hidden\_dim}$<$64.

\subsection{7.8 Architecture Hardening: Online Precision Recalibration
          (\texttt{models/degradation\_controller.py})}

\texttt{OnlinePrecisionMonitor}: rolling window (window\_size=500, min\_samples=200);
\texttt{ingest\_feedback()} rejects out-of-order timestamps; \texttt{estimate\_precision()}
returns P@50 from top-50 scored accounts in window (or \texttt{None} $<$ 200 samples).

\texttt{EnhancedConceptDriftDetector}: per-path \texttt{PrecisionDriftDetector} instances
(no cross-path contamination); \texttt{classify\_drift()} returns:
\textit{precision\_degraded} (precision $<$ budget, regardless of KL),
\textit{benign\_shift} (KL $>$ 0.15 but precision $\geq$ budget),
\textit{no\_drift} (otherwise).

\texttt{EnhancedAdaptivePrecisionBudget}: blended precision $= 0.7\times p_\text{online}
+ 0.3\times p_\text{shadow}$; tightens $+$0.05 after 5 consecutive $>$ base\,+\,0.10;
relaxes $-$0.02 after 3 consecutive within 0.05 of base; output clamped to $[0.55,\,0.75]$.

\subsection{7.9 Architecture Hardening: Resource Manager
          (\texttt{models/resource\_manager.py --- ResourceManager})}

\begin{itemize}[noitemsep,leftmargin=*]
  \item \texttt{allocate\_penalty\_buffers(mode)}: allocates distinct memory buffers
        (symmetric: 2\,MB; learnable: 8\,MB) for each SCC penalty mode. Req.~7.1.
  \item \texttt{route\_topology\_pipeline(use\_gnn)}: routes to GNN pipeline (80\,MB,
        120\,ms, 4 processing units) or manual pipeline (2\,MB, 5\,ms, 1 unit). Req.~7.2.
  \item \texttt{release\_degraded\_resources(disabled\_components)}: frees component memory
        on degradation events. Req.~7.4.
  \item \texttt{atomic\_model\_swap(new\_weights)}: thread-safe atomic weight replacement
        (deep-copy + lock) without interrupting scoring. Req.~7.6.
  \item \texttt{utilization\_report()}: generates \texttt{ResourceUtilizationSnapshot}
        records with memory delta, computation time, and active-unit count per event. Req.~7.3, 7.5.

\section{9.\ What Aspect(s) of the Invention Need(s) Protection?}
% ══════════════════════════════════════════════════════════════════════

\subsection{Claim 1 --- Temporal-Decay PageRank with Burst-Velocity Amplification
            and Directional SCC Penalty}
\begin{enumerate}[label=1.\arabic*,leftmargin=2.5em,noitemsep]
  \item Embedding $\exp(-\lambda\times\text{EdgeAge})$ \emph{inside} the power iteration
        (not as pre-processing on time-binned snapshots).
  \item Burst-velocity amplification: per-edge multiplier $\geq$1.0 for sender nodes
        with transaction surges within a configurable window.
  \item Log-amount normalisation replacing raw amounts to prevent single-transaction dominance.
  \item Directional SCC penalty: asymmetric dampening distinguishing collectors (0.25$\times$),
        distributors (0.50$\times$), and balanced nodes (0.375$\times$).
  \item Dormant node suppression: capping scores at 0.1$\times$max when all incident
        edges exceed a configurable age threshold.
  \item (New) \texttt{LearnableSCCPenalty} MLP: trainable replacement for the static
        3-archetype penalty, retrained without code changes, updated atomically.
\end{enumerate}
\textit{Scope:} Method claims for the combination applied to directed financial
transaction graphs with timestamped, amount-weighted edges.

\subsection{Claim 2 --- Topology-Conditioned Multiplicative Attention Gate
            for Heterogeneous Signal Fusion}
\begin{enumerate}[label=2.\arabic*,leftmargin=2.5em,noitemsep]
  \item Per-account 5- or 6-dimensional \texttt{TopologyVector} from 2-hop ego-network.
  \item \texttt{TopologyAttentionGate}: \emph{multiplicative} gating on the weight
        dimension (not additive attention on feature space); gate amplifier
        $\in[1.0,\,3.0]$.
  \item Complete per-account weight pipeline: TopologyVector $\to$ gate $\to$
        clamped weights $\to$ fused score; computed independently per account per batch.
  \item Isolation safety floor: $w_\text{ml}\geq 0.70$ when ego-network $<$3 nodes.
  \item Complete 6-condition fallback chain with static weights (0.70, 0.15, 0.15).
  \item (New) \texttt{TopologyEmbeddingNetwork}: GNN-based learned ego-network embedding
        (2$\times$GCNConv, 16-d) replacing manual \texttt{TopologyVector}.
  \item (New) \texttt{IsolationDetector}: learned continuous isolation function replacing
        hard-coded threshold; monotonically non-decreasing $w_\text{ml}$.
  \item (New) \texttt{DeepTopologyAttentionGate}: $\geq$2 hidden layers, dropout, deeper
        multiplicative gating.
\end{enumerate}

\subsection{Claim 3 --- Precision-Budget-Constrained Adaptive Degradation Controller}
\begin{enumerate}[label=3.\arabic*,leftmargin=2.5em,noitemsep]
  \item Pre-computed execution path routing table with P@K annotations; selection
        constrained by precision budget (not latency/availability).
  \item \texttt{PrecisionDriftDetector}: EMA-smoothed KL-divergence for proactive
        precision degradation detection.
  \item \texttt{AdaptivePrecisionBudget}: self-calibrating minimum P@K threshold;
        tightens/relaxes based on rolling shadow evaluation history.
  \item Health-monitoring state machine: 2-cycle DEGRADED detection, 3-cycle HEALTHY
        recovery, flap protection, 30-min stale queue policy, $\leq$500\,ms routing SLA.
  \item Drift KL-divergence as additional input to budget computation.
  \item (New) \texttt{OnlinePrecisionMonitor}: live P@50 from rolling labeled feedback
        (window=500, min=200 samples); timestamp ordering validation.
  \item (New) \texttt{EnhancedConceptDriftDetector}: KL + precision jointly classify
        \textit{no\_drift} / \textit{benign\_shift} / \textit{precision\_degraded}
        with per-path isolation.
  \item (New) \texttt{EnhancedAdaptivePrecisionBudget}: 0.7$\times$online +
        0.3$\times$shadow blending; tightening/relaxation with consecutive-estimate logic.
\end{enumerate}
